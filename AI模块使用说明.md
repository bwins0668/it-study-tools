# AI 模块使用说明

本说明面向 Study Tools 普通用户，介绍如何使用 AI 学习中心、AI 导师、学习教练和 AI 自适应出题功能。

## 1. 打开 AI 学习中心

启动 Study Tools 后，在顶部导航栏找到 **AI 学习** 按钮。

点击后会打开 **AI 学习中心**，里面有四个入口：

- **AI 导师**：针对当前课程、代码、输入和运行结果进行答疑。
- **学习教练**：查看五科掌握度、今日计划、薄弱点和成绩趋势。
- **自适应出题**：按当前科目、知识点和难度生成练习题。
- **Provider 设置**：配置 Gemini、OpenAI 或 Ollama。

## 2. 配置 AI Provider

首次使用 AI 回答或 AI 出题前，请先打开 **AI 学习中心 → Provider 设置**。

支持三种 Provider：

- **Gemini**：默认模型 `gemini-2.5-flash`
- **OpenAI**：默认模型 `gpt-5-mini`
- **Ollama**：使用本机已安装的第一个模型，也可以手动填写模型名

### API Key 说明

如果电脑环境变量中已经配置了密钥，可以不填写 API Key。

如果在界面中填写 API Key：

- 只保存在当前浏览器会话中。
- 关闭浏览器或重新打开程序后可能需要重新填写。
- 不会写入 SQLite 数据库。
- 不会写入日志。
- 不会保存到 `localStorage`。

Ollama 不需要 API Key，但需要先在本机启动 Ollama 服务。

默认地址：

```text
http://127.0.0.1:11434
```

## 3. 使用 AI 导师

打开 **AI 学习中心 → AI 导师**。

AI 导师会自动读取当前学习上下文，包括：

- 当前科目
- 当前课程或题目
- SQL / Java / Python 代码
- 输入内容
- 运行结果
- 错误信息

你可以直接输入问题，例如：

```text
为什么我的 SQL 没有结果？
帮我解释这段 Python 代码。
我的 Java 报错是什么意思？
这一课的重点是什么？
```

### 快捷辅导按钮

AI 导师面板中提供四个快捷按钮：

- **分析错误**：根据当前运行错误分析原因。
- **解释代码**：解释当前代码的执行逻辑。
- **逐步执行**：按步骤说明代码如何运行。
- **错题讲解**：提交考试或练习后，解释错题原因。

### 三级提示

AI 导师提供渐进式提示：

- **提示 1：只给思路**  
  只提示方向，不给答案。

- **提示 2：给出结构**  
  提供解题框架或关键步骤。

- **提示 3：参考代码**  
  在需要时给出接近答案的参考写法。

建议先用提示 1，再逐步升级，避免过早看到答案。

## 4. 使用学习教练

打开 **AI 学习中心 → 学习教练**。

学习教练主要显示：

- 五科综合掌握度
- SQL / Java / Python / IT Passport / SG 分科掌握度
- 今日学习计划
- 优先复习内容
- 最近成绩趋势
- 有效练习次数
- 连续学习天数

学习教练的核心统计由本地规则计算，即使没有网络、没有 API Key，也能正常显示。

完成今日计划后，点击计划右侧的 **完成** 按钮，系统会自动记录学习事件并更新统计。

## 5. 使用 AI 自适应出题

打开 **AI 学习中心 → 自适应出题**。

选择：

- 科目
- 知识点
- 难度

然后点击 **生成并自动校验**。

AI 生成题目后，系统会做质量检查：

- 检查题目结构是否完整。
- 进行第二轮 AI 审核。
- SQL / Java / Python 题会验证标准答案是否能实际运行。
- IT Passport / SG 题会检查是否为四选一、答案唯一、解释完整。

合格题目会进入临时 AI 题库，不会修改原始题库文件。

### 编程题如何作答

SQL / Java / Python 生成题可以载入对应沙盒。

载入后：

1. 在沙盒中补全代码。
2. 点击运行。
3. 系统会根据标准答案或期望输出自动评分。

### 选择题如何作答

IT Passport / SG 生成题会显示四个选项。

点击选项后，系统会：

- 标记正确答案。
- 显示解释。
- 记录本次练习结果。

## 6. 考试期间的 AI 限制

正式模拟考试进行中，AI 会自动禁用。

禁用范围包括：

- AI 导师问答
- 快捷提示
- AI 出题

提交试卷后，可以重新使用 AI 查看错题讲解和变式练习。

这样可以避免考试过程中提前泄露答案。

## 7. 数据与隐私

Study Tools 是单用户、本机便携系统。

学习数据保存在本机 SQLite 数据库中，主要包括：

- 学习事件
- 课程完成记录
- 答题结果
- 代码运行结果
- 耗时
- 错误类型
- 提示次数
- AI 题目生成记录
- 每日计划

AI 相关注意事项：

- API Key 不写入数据库。
- API Key 不写入日志。
- API Key 不保存到长期浏览器存储。
- AI 题目与官方真题分开统计。
- AI 不会自动执行任意返回代码。
- 只有通过校验流程的标准答案会被用于本地验证。

## 8. 常见问题

### 提示缺少 API Key

请打开 **AI 学习中心 → Provider 设置**，填写当前 Provider 对应的 API Key。

如果使用环境变量配置密钥，请确认启动程序前环境变量已经生效。

### Ollama 显示未启动

请先启动 Ollama 服务，并确认地址为：

```text
http://127.0.0.1:11434
```

同时确认已经安装至少一个本地模型。

### 模型不存在

请检查 Provider 设置中的模型名称。

可以先留空，使用系统默认模型。

### AI 回答很慢

可能原因：

- 网络较慢。
- 当前模型响应慢。
- Ollama 本地模型较大。
- 生成题目正在进行执行校验。

可以尝试更换速度更快的模型。

### 没有网络还能用吗

可以。

没有网络或没有 API Key 时：

- 原有课程学习可用。
- SQL / Java / Python 沙盒可用。
- CBT 考试可用。
- 学习教练本地统计可用。

但 AI 问答和 AI 生成题目需要可用的 Provider。

## 9. 推荐使用方式

日常学习建议：

1. 先阅读课程内容。
2. 自己完成题目或代码。
3. 运行后如果出错，使用 **分析错误**。
4. 不会做时先用 **提示 1**。
5. 仍卡住再用 **提示 2**。
6. 最后再看 **提示 3** 或参考代码。
7. 每天打开 **学习教练** 查看今日计划。
8. 对薄弱知识点使用 **自适应出题** 追加练习。
